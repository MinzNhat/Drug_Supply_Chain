# Test Output Format

All end-to-end execution records are centralized in this folder.

## Naming Convention

- `test_001_prerequisites.txt`
- `test_002_bring-up.txt`
- `test_003_runtime-e2e.txt`
- `test_004_geo-flow-e2e.txt`
- `test_005_transfer-batch.txt`
- `test_006_teardown.txt`

Files are generated by `./scripts/test-all.sh`.

Single-mode runs also use canonical IDs (`002` for `up`, `003` for `test`, `004` for `geo`, `005` for `transfer`, `006` for `down`) to avoid duplicate historical naming.

## File Schema

Each file contains:

1. `description`: What this step validates.
2. `input`: Exact command executed.
3. `started_at`: UTC timestamp.
4. `output`: Raw stdout/stderr captured from the command.
5. `ended_at`: UTC timestamp.
6. `status`: `SUCCESS` or `FAILED`.
